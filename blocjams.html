DESIGN PROCESS

RESEARCH

I. COMPETITIVE ANALYSIS

By conducting SWOT analyses for WeHeartIt, Pinterest and Reddit, I know that sharing contents without restrictions could lead to many issues, such as spam and quality control. Yet, the advantage is that it provides an open door for the company to reach more potential customers who show particular interests that match them.

Icons of WeHeartIt, Pinterest and Reddit

II. USER SURVEY

The key information I am interested the most are “Why do people use cloud storage and organizing app” and “What is/are the frustration(s)?” because it helped me to maintain a general direction. From the survey, we know that people using cloud storage and organizing app for 3 main reasons. The top reasons are:

A screenshot of survey result

“It’s common and convenient to work with the others” and “accessibility that you can get the files more than one place”.

I need to ensure the workflows of sharing-related features, such as download and share link are effective. Additionally, ALL of the participants are using mobile to connect to cloud storage service, while 94.1% of them are using both computer and mobile to enjoy the cloud service. Therefore, besides doing a desktop version for the app, it is necessary to build a mobile version too.

The second highest vote for the reason to use a cloud storage app is “to save space on my mobile/computer”.

It could be seen as another group of audience who uses the cloud storage app for personal use besides collaborating with people.

At last, “bad organization experience” comes to the most frustrating part from the users. It affects both groups of people who use the cloud storage app for their own uses and those who use it for collaboration. So, the organizing flow should be worth putting the most effort.

PERSONA

Building persona gives me the idea who the users are and that would be useful to make sure non-designers are on the same page for collaboration. There are two main types of potential users that we learned from the survey.

One is the people who want to save files for back-up or their own uses, requiring a lot of storage capacity and well-developed organizing feature, so that they can manage it and save space.

The other one is using it to collaborate with people. They expect to access the files anywhere and share them effectively to people or social media. Here below are some examples.

STYLE GUIDE

I. COLOR

It is common to use blue to represent profession and security, but the brand was going to be more casual, so I chose a softer color with lower chroma as the primary color.

Another sharper color-yellow, which is used for drawing the user’s attention and driving them to click. I used yellow for the content and buttons on the Account Upgrade section to differentiate it from other pages because I want the user to feel that is a special part that we treat the valuable customer differently.

Color section of Style Guide

For less important information or button such as ‘browse’ to upload a file, I would use green to support it, so that yellow still stands out among these colors.

Account Upgrade page

II. TYPOGRAPHY

Using Montserrat typeface for headings is modern and clean as it provides enough spacing between each character, making it easy to read, especially most of the headings I used are single-lined. For the body, I chose Libre Baskerville because serif type can direct the user’s reading. Again, the character spacing is not packed, resulting in comfortable reading.

Typography section of Style Guide

USABILITY AND PREFERENCE TESTS

In the final round of usability test, I tested sign-in, sign-up, sign-out, upload, download, share, move, upgrade an account, update the profile.

The feedbacks are “create a new folder” button on the dashboard is missing”, “MY PROFILE button is a bit outdated and not necessary” and “create a sub-folder quickly while moving a file”.

So, I added a “create a new folder” icon on the dashboard, so that this feature is not only limited while moving a file. Then, I simplified the way of updating “my profile” by adding a pen icon next to the profile picture to allow the user to edit it. At last, I created an optional form for the user to create a sub-folder at the same time while moving a file.

An optional form creating a sub-folder is created

PREFERENCE TEST I: TYPOGRAPHY

The difference of typography here is switching the font faces for heading and body.

It is using Libre Baskerville for heading and Montserrat for body.
It is using Montserrat for heading and Libre Baskerville for body.

100% of the participants go for the second one because the concern for the readability on a tablet, computer and mobile, the softer look would make it clearer to read. Also, they think Montserrat looks modern and professional in a heading.

PREFERENCE TEST II: COLOR

Action bar in blue
Action bar in green

Although 88% of participants go for the one with a blue pop-up box due to consistency with the theme color. I took the comments from both sides and decided to use the green one because the contrast is very important to tell the user that they have selected the file and do some actions with it.

This light up step is very important because I skipped placing all the buttons on the top right corner. Participants reflected that it was annoying to go back and forth to find the buttons especially when they already scrolled down to select the files.

Another point is that the first button of “select” is redundant. Users should be able to select the files by clicking them. These are the takeaways from the first round of usability test. Therefore, the action buttons will only pop up when the user selects something, and of course, I took away the “select” button since the user can directly click on the file.

Origianl wireframe of the dashboard

PREFERENCE TEST III: LOGO

I modified the logo with 2 more options for the usability test. I thought providing the name would make it easier for the audience to understand what the app is by just looking at it.

Three logo options

However, my assumption was wrong. The result of the usability test is, people would prefer the first one. Respondents said that it was to the point, unique and adorable.

I decided to use the icon only without wording because my branding tends to be friendly and casual. The other one would look more serious comparing to the cloud icon itself.

CONCLUSION

SOLUTION

It solved the problems of overwhelming options for each event for the user. The action buttons like download and share icons would only appear when the user selects a file, and they do not need to go back to the very top to click the action buttons.

Also, I tried to reduce the steps of action, such as sharing, it directly goes to the sharing link to make it convenient for the user to copy it.

IMPROVEMENT?

I think users may not know that they can select more than one file at a time and unselect it by clicking on it again. A simple tutorial section can be added after a user sign up to improve the user experience.

Another minor thing I can think of is, the look of thumbnail for the files can be unified, e.g. MP4, PDF, document and notes. So that the user can be able to understand the file types instantly.
